🤔 Що це взагалі таке?

Уявіть, що Claude Code — це ваш цифровий помічник-програміст. «Голий» помічник вміє багато, але кожен проєкт особливий: одному потрібна допомога з дизайном, іншому — з документами, третьому — з аналізом даних.

Щоб не носити з собою «всі знання світу», помічник вчиться «навичкам» (скілам) і встановлює «розширення» (плагіни). Це як додатки на телефоні: завантажили — отримали нову суперсилу.

🎯
Скіл (Skill)
Спеціальна навичка — автоматично «вмикається», коли Claude бачить релевантну задачу.
🔌
Плагін (Plugin)
Пакет з кількох скілів і команд. Зазвичай має тематику (дизайн, документи, веб-доступ).
🛰️
MCP-сервер
Окрема програма, що дає Claude нові «руки» — підключення до зовнішнього сервісу (Notion, Telegram, Google).
⚡
CLI
Утиліта в терміналі (наприклад, gh), якою Claude користується через Bash. Часто пара до плагіна.

🛠️ Як встановити плагін?

Встановлення складається з трьох простих кроків. Просто скопіюйте команди в чат з Claude Code:

📥 Встановлення в три кроки
# 1. Додати маркетплейс (один раз)
/plugin marketplace add <автор>/<репозиторій>

# 2. Встановити плагін
/plugin install <ім'я-плагіна>

# 3. Готово! Перевірити встановлені:
/plugin list

💡 Більшість плагінів від Anthropic вже доступні без додавання маркетплейсу — просто пишіть /plugin install ім'я.

Playwright
⭐ 88 000 зірок
Браузерна автоматизація від Microsoft — точний контроль через скрипти
🔵 Через MCP / CLI microsoft/playwright

Промисловий стандарт для тестування і автоматизації веб-сторінок. Дозволяє Claude управляти Chromium, Firefox і WebKit через єдиний API: клікати, заповнювати форми, робити скріншоти, чекати на завантаження, перевіряти стан сторінок.

На відміну від Browser Harness, Playwright запускає свій ізольований браузер. Найкраще пасує коли потрібно: писати E2E-тести (з повторюваним результатом), скрейпити публічні сторінки, генерувати скріншоти для документації, автоматизувати на CI/CD без головного браузера користувача.

# Або як CLI глобально:
npm install -g @playwright/cli
Як використовувати: «протестуй цю форму», «зроби E2E тест для логіну», «перевір що сторінка відкривається у мобільному вигляді», «зроби скріншот сторінки X».

Browser Harness
⭐ 11 000 зірок
Прямий контроль вашого Chrome — без логінів і авторизацій
🔵 Через CLI browser-use/browser-harness

Дозволяє Claude управляти вашим живим Chrome — кликати кнопки, заповнювати форми, робити скріншоти, читати сторінки за вашими логінами. Не запускає окремий браузер — підключається до того, що вже відкрито.

Корисно коли потрібно автоматизувати рутину в авторизованих сервісах: «зайди в мою адмінку і витягни список продажів», «заповни цю довгу форму», «зроби expert-режим у Notion». Працює на рівні браузера, тож не вимагає API.

🤔 Playwright vs Browser Harness — коли що використовувати?
  • Playwright — коли потрібен стабільний скрипт: тести, скрейпери на CI, повторювані операції на чистому браузері.
  • Browser Harness — коли потрібен живий контроль: робота в авторизованих сервісах (адмінки, банки, корпоративні портали), де ви вже залогінені у своєму Chrome.

💡 Просте правило: треба ввести логін і пароль? — Browser Harness. Треба написати тест? — Playwright.

🔵 Через команди Shawnchee/caveman-skill

Викидає всі «зайві» слова з відповідей Claude — артиклі, ввічливості, hedging-фрази. Зберігає 100% технічного змісту, але економить ~75% токенів. Має рівні: лайт, повний, ультра.

Корисно для довгих сесій, коли контекст обмежений. Для кодів, комітів, попереджень про безпеку — автоматично перемикається в нормальний режим (auto-clarity), щоб уникнути двозначності.

📥 Встановлення
/plugin marketplace add Shawnchee/caveman-skill
/plugin install caveman
Корисні команди:
  • /caveman lite|full|ultraПеремкнути рівень стиснення
  • /caveman-statsСкільки токенів зекономлено
  • /caveman-helpДовідник усіх команд
  • stop cavemanПовернутись до нормальної мови

🛰️ MCP-сервери (поза плагінами)

MCP-сервер — окрема програма, яку Claude підключає для нових можливостей. Часто йдуть із плагінами, але деякі варто поставити окремо.

🪡 Stitch (Google) ⭐ Google product

Експериментальний MCP від Google для генерації UI-макетів. Скажіть «створи мені дизайн ландінгу для X» — отримаєте структурований макет, який можна одразу імпортувати у Figma або скопіювати у Tailwind-код.

claude mcp add stitch https://stitch.googleapis.com/mcp --transport http

🔗 n8n MCP ⭐ workflow automation

Підключення до n8n — open-source no-code платформи автоматизації (~95k зірок саме у n8n). Через MCP Claude може створювати, редагувати, запускати ваші workflow: «створи мені ланцюжок: Telegram повідомлення → ChatGPT → запис у Google Sheet».

npm install -g n8n-mcp
claude mcp add n8n npx n8n-mcp

📦 Офіційний реєстр MCP-серверів ⭐ 85 000

Колекція офіційних MCP-серверів від команди MCP: memory (вбудована памʼять), filesystem, git, github, sqlite, postgres, puppeteer, brave-search та інші. Якщо щось вам потрібне — спочатку перевірте чи воно вже є тут.

# Приклад — підключити memory сервер:
npm install -g @modelcontextprotocol/server-memory
claude mcp add memory npx @modelcontextprotocol/server-memory

⌨️ CLI-інструменти у тандемі з Claude

Claude може запускати будь-який CLI через Bash. Ось ті, що варто мати глобально — Claude буде використовувати їх автоматично.

🐙 gh — GitHub CLI офіційний

Робота з GitHub з командного рядка: PR, issues, releases, workflows, репозиторії. Claude використовує його для перегляду PR, створення issues, перевірки CI-статусу. Без цього багато github-сценаріїв не запрацюють.

brew install gh
gh auth login

📊 gws — Google Workspace CLI ⭐ 26 000

Робота з Drive, Sheets, Docs, Gmail, Calendar з терміналу. Claude може створювати документи, читати таблиці, надсилати листи від вашого імені.

npm install -g @googleworkspace/cli
gws auth login

▲ vercel — Vercel CLI ⭐ 15 000

Деплой проєктів на Vercel. Claude уміє за командою «задеплой» зібрати білд, налаштувати env-змінні, прив'язати домени.

npm install -g vercel
vercel login

🔥 firecrawl — Firecrawl CLI пара до плагіна

Командний інтерфейс до Firecrawl для скриптів і CI. Працює без MCP — корисно коли треба запустити скрейпінг у скрипті чи cron.

npm install -g firecrawl-cli

🎭 playwright-cli пара до плагіна

CLI до Playwright — швидкий запуск автотестів і скрейпінг скриптами без MCP. Для CI/CD або разових перевірок.

npm install -g @playwright/cli

⚡ Рекомендований стек для початківця

Не намагайтесь поставити все одразу. Ось перевірений порядок:

  1. Базовий мінімум: Claude Code + Superpowers + Anthropic Skills. Цього вистачить на 80% задач.
  2. CLI-інструменти: gh, gws, vercel — встановити глобально, авторизуватись.
  3. Для коду: code-review, frontend-design, claude-code-setup, skill-creator з офіційного маркетплейсу.
  4. Для веб-доступу: Firecrawl + Context7. Інтернет і свіжа документація.
  5. Для браузерної автоматизації: Playwright (тести, скрейпери) + Browser Harness (логіни, адмінки).
  6. Для пам'яті: Claude Memory. Щоб не пояснювати одне й те саме щоразу.
  7. Якщо робите інтерфейси: UI/UX Pro Max + Lazyweb + плагін design.
  8. Економія токенів: Caveman (увімкнути full).
  9. Мобільний контроль: Telegram Supercharged.
